# iOS Project Configuration

Build infrastructure for a Crux iOS shell using XcodeGen and cargo-xcode.

## Directory Layout

```
{app-dir}/
    Cargo.toml              # Rust workspace (already exists from core-writer)
    shared/                 # Rust shared crate (already exists)
        shared.xcodeproj/   # Generated by cargo-xcode
    iOS/
        project.yml         # XcodeGen project definition
        Makefile            # Build automation
        generated/          # UniFFI-generated Swift bindings (gitignored)
        {AppName}/          # Swift source files
            {AppName}App.swift
            Core.swift
            ContentView.swift
            Views/
                LoadingScreen.swift
                MainScreen.swift
                ErrorScreen.swift
```

## XcodeGen `project.yml`

This YAML file defines the Xcode project. Run `xcodegen` to generate the
`.xcodeproj` from it.

### Template

```yaml
name: {AppName}
projectReferences:
  Shared:
    path: ../shared/shared.xcodeproj
packages:
  VectisDesign:
    path: ../../../design-system/ios
options:
  bundleIdPrefix: com.vectis.{app-name}
  deploymentTarget:
    iOS: "17.0"
attributes:
  BuildIndependentTargetsInParallel: true
targets:
  {AppName}:
    type: application
    platform: iOS
    sources:
      - {AppName}
      - path: ../shared/src/shared.udl
        buildPhase: sources
        optional: true
    dependencies:
      - target: Shared/shared-staticlib
      - package: VectisDesign
    info:
      path: {AppName}/Info.plist
      properties:
        UISupportedInterfaceOrientations:
          - UIInterfaceOrientationPortrait
          - UIInterfaceOrientationLandscapeLeft
          - UIInterfaceOrientationLandscapeRight
        UILaunchScreen: {}
    settings:
      base:
        SWIFT_VERSION: "6.0"
        SWIFT_STRICT_CONCURRENCY: complete
        OTHER_LDFLAGS: ["-w"]
        SWIFT_OBJC_BRIDGING_HEADER: generated/sharedFFI.h
        ENABLE_USER_SCRIPT_SANDBOXING: "NO"
      configs:
        Debug:
          PRODUCT_BUNDLE_IDENTIFIER: com.vectis.{app-name}.debug
        Release:
          PRODUCT_BUNDLE_IDENTIFIER: com.vectis.{app-name}
    buildRules:
      - name: Generate UniFFI Bindings
        filePattern: "*.udl"
        script: |
          #!/bin/bash
          set -e
          if [ "$ACTION" == "indexbuild" ]; then
            echo "Skipping UDL processing during indexing."
            exit 0
          fi
          if [ "$ENABLE_PREVIEWS" = "YES" ]; then
            echo "Skipping UDL processing during preview builds."
            exit 0
          fi
          cd "${INPUT_FILE_DIR}/.."
          "${BUILD_DIR}/${CONFIGURATION}/uniffi-bindgen" \
            generate "src/${INPUT_FILE_NAME}" \
            --language swift \
            --out-dir "${PROJECT_DIR}/generated"
        outputFiles:
          - $(PROJECT_DIR)/generated/$(INPUT_FILE_BASE).swift
          - $(PROJECT_DIR)/generated/$(INPUT_FILE_BASE)FFI.h
        runOncePerArchitecture: false
```

### Adapting the Template

- Replace `{AppName}` with the PascalCase app name (e.g., `TodoApp`).
- Replace `{app-name}` with the kebab-case app name (e.g., `todo-app`).
- The `packages.VectisDesign.path` is relative from the `iOS/` directory to
  `design-system/ios/`. Adjust based on the app's location in the repo.
- If the app does not use UniFFI `.udl` files (uses attribute-based UniFFI
  only), remove the `buildRules` section and the `.udl` source entry.

## Makefile

Build automation for compiling the Rust library and generating the Xcode project.

### Template

```makefile
.PHONY: all setup build clean xcode rust-lib

SHARED_DIR := ../shared
TARGETS := aarch64-apple-ios aarch64-apple-ios-sim

all: setup build

setup: rust-lib xcode

# Build the Rust shared library for iOS targets
rust-lib:
	@echo "Building Rust shared library..."
	@for target in $(TARGETS); do \
		cargo build --manifest-path $(SHARED_DIR)/Cargo.toml \
			--target $$target --release --features uniffi; \
	done

# Generate Xcode sub-project for the shared library
cargo-xcode:
	@echo "Generating Xcode project for shared library..."
	@cd $(SHARED_DIR)/.. && cargo xcode

# Generate Xcode project from project.yml
xcode: cargo-xcode
	@echo "Generating iOS Xcode project..."
	@xcodegen

# Build the iOS app
build:
	@xcodebuild build \
		-project {AppName}.xcodeproj \
		-scheme {AppName} \
		-destination 'platform=iOS Simulator,name=iPhone 16' \
		-configuration Debug \
		CODE_SIGNING_ALLOWED=NO \
		2>&1 | xcbeautify

# Clean build artifacts
clean:
	@xcodebuild clean \
		-project {AppName}.xcodeproj \
		-scheme {AppName} \
		2>&1 | xcbeautify
	@rm -rf generated/
```

## Info.plist

The `Info.plist` is generated by XcodeGen from the `info.properties` in
`project.yml`. For additional entries, add them to the properties map:

```yaml
info:
  path: {AppName}/Info.plist
  properties:
    UILaunchScreen: {}
    UISupportedInterfaceOrientations:
      - UIInterfaceOrientationPortrait
    NSAppTransportSecurity:
      NSAllowsArbitraryLoads: true  # only for development
```

## .gitignore

Add to the app's `.gitignore`:

```
# Xcode
*.xcodeproj/
*.xcworkspace/
DerivedData/
build/

# Generated UniFFI bindings
iOS/generated/

# Build artifacts
iOS/*.xcodeproj
```

Keep `project.yml` and `Makefile` tracked -- these are the source of truth
for the Xcode project configuration.

## Swift Version

Target Swift 6.0 with strict concurrency checking:

```yaml
settings:
  base:
    SWIFT_VERSION: "6.0"
    SWIFT_STRICT_CONCURRENCY: complete
```

## Deployment Target

Minimum iOS 17.0 to ensure availability of:
- `NavigationStack` (iOS 16+)
- `@Observable` macro (iOS 17+)
- Modern SwiftUI APIs (`onChange` with two parameters, etc.)

## Code Signing

For development and CI builds, disable code signing:

```yaml
settings:
  base:
    CODE_SIGNING_ALLOWED: "NO"
```

For distribution builds, configure signing identity and provisioning profile
in the Release configuration.
