## Why

The `opsx_todo` Crux application was generated by the core-writer skill and
passes `cargo check`, `cargo test` (22 tests), and `cargo clippy` cleanly.
A systematic code review using the core-reviewer skill found 3 Critical,
6 Warning, and 2 Info findings across structural, logic, and quality checks.

Two mechanical issues (CRX-001: missing `render()` after state mutations in
`ConnectSse` and `Navigate` from Error) were auto-fixed during the review
cycle. The remaining findings are design-level issues requiring architectural
decisions.

## What Changes

### Critical

- **[CRX-003] `PendingOp::Delete` missing conflict-resolution data** --
  `Delete(String)` carries only an item ID with no `deleted_at` timestamp.
  When an SSE update arrives for a locally-deleted item, the conflict
  resolution cannot determine temporal ordering.

- **[CRX-004 / LOG-002] Operation coalescing missing on destructive ops** --
  `DeleteTodo` and `ClearCompleted` blindly push `PendingOp::Delete` without
  checking whether the item only exists as a pending `Create` (never synced).
  This causes unnecessary 404 errors from the server.

- **[LOG-003] Concurrent operation conflicts** -- SSE `item_deleted` handler
  removes ALL pending ops for an item without checking `syncing_id`, which
  could lose a valid mutation queued after the sync started.

### Warning

- **[CRX-007] IDs and timestamps generated inside `update()`** --
  `AddTodo`, `EditTitle`, and `ToggleTodo` use `model.next_id += 1` and
  `Utc::now()` directly, making the core non-deterministic for testing.

- **[CRX-009 / LOG-008] Missing edge-case tests** -- No tests for: SSE
  during in-flight sync, conflict resolution (server-wins / local-wins),
  Create-then-Delete coalescing, rapid toggles, ClearCompleted with no
  completed items.

- **[LOG-005] Silent data loss on corrupted state** --
  `serde_json::from_slice(&bytes).unwrap_or_default()` silently replaces
  corrupted persisted state with empty defaults.

- **[LOG-006] Duplicate pending ops on rapid actions** -- Rapid toggles of
  the same item produce duplicate `PendingOp::Update` entries without
  coalescing.

- **[GEN-005] `StateSaved` error silently swallowed** --
  `StateSaved(Ok(_) | Err(_)) => Command::done()` ignores save failures.

### Info

- **[CRX-008] ViewModel uses pre-formatted strings** -- `active_count` and
  `sync_status` are `String` fields with core-formatted text. Should be typed
  values for shell-side localization.

## Impact

- **Data model change**: `PendingOp::Delete` gains a `deleted_at` field,
  affecting `PersistedState` serialization (migration needed for existing
  stored data).
- **Event signature change**: `AddTodo`, `EditTitle`, `ToggleTodo` may gain
  `id` and `timestamp` parameters from the shell.
- **Shell contract change**: `TodoListView` field types may change from
  `String` to `usize` / enum.
- **Files affected**: `shared/src/app.rs` (all changes), `spec.md` (if
  event signatures change).
